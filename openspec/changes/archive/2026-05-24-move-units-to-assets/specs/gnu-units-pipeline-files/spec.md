## ADDED Requirements

### Requirement: GNU Units source files live in assets/units/
The GNU Units source data files (`definitions.units` and any companion `.units`
files) SHALL be located in `assets/units/` at the repository root.  The
associated `LICENSE.md` and `README.md` for those files SHALL also reside in
`assets/units/`.

#### Scenario: Import tool reads from assets/units/
- **WHEN** a developer runs `dart run tool/import_gnu_units.dart`
- **THEN** the tool reads `definitions.units` from `assets/units/definitions.units`
  (relative to the repository root)
- **AND** no files from `tool/gnu_units/` are required

---

### Requirement: JSON pipeline artifacts live in assets/units/
The three JSON files that make up the unit pipeline intermediate and merged
databases SHALL be located in `assets/units/`:

- `units-parsed.json` — generated output of the importer; SHALL NOT appear
  under `lib/`
- `units-supplementary.json` — manually curated overlay data; SHALL NOT appear
  under `lib/`
- `units.json` — merged database generated by the codegen tool; SHALL NOT
  appear under `lib/`

#### Scenario: Importer writes units-parsed.json to assets/units/
- **WHEN** a developer runs `dart run tool/import_gnu_units.dart`
- **THEN** the tool writes its output to `assets/units/units-parsed.json`
- **AND** no file is written to `lib/core/domain/data/`

#### Scenario: Codegen reads from and writes to assets/units/
- **WHEN** a developer runs `dart run tool/generate_predefined_units.dart`
- **THEN** the tool reads `assets/units/units-parsed.json` and
  `assets/units/units-supplementary.json`
- **AND** writes the merged database to `assets/units/units.json`
- **AND** writes the generated Dart source to
  `lib/core/domain/data/predefined_units.dart` (unchanged location)

---

### Requirement: Generated Dart source remains in lib/core/domain/data/
The generated Dart source files (`predefined_units.dart`,
`builtin_functions.dart`) SHALL remain in `lib/core/domain/data/`.  Only
non-Dart pipeline data files are relocated.

#### Scenario: Dart source location is unchanged after reorganization
- **WHEN** the codegen tool runs successfully
- **THEN** `predefined_units.dart` is written to
  `lib/core/domain/data/predefined_units.dart`
- **AND** no Dart source file is placed under `assets/`

---

### Requirement: Pipeline data files are not declared as Flutter runtime assets
The `.units` source files and the JSON pipeline artifacts SHALL NOT be listed
under `flutter.assets` in `pubspec.yaml`.  They are codegen-time inputs/outputs
only and MUST NOT be bundled into the app binary.

#### Scenario: pubspec.yaml has no asset declaration for pipeline files
- **WHEN** a developer inspects `pubspec.yaml`
- **THEN** none of `*.units`, `units-parsed.json`, `units-supplementary.json`,
  or `units.json` appear in the `flutter.assets` list
