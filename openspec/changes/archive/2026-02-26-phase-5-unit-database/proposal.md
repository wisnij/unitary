## Why

Phase 5 of the implementation plan requires completing the unit database with all
required categories (volume, area, speed, pressure, energy, digital storage).
Rather than defining units piecemeal in Dart by hand, we need a systematic
import pipeline sourced from the authoritative GNU Units definitions file, with
checked-in JSON intermediates that can be hand-annotated and re-imported as
support for new unit types expands.

## What Changes

- Add `tool/import_gnu_units.dart` + `tool/import_gnu_units_lib.dart`: parses a
  GNU Units definitions file (`tool/gnu_units/definitions.units`) and writes
  importer-owned fields to `lib/core/domain/data/units-parsed.json`. The
  importer does not read or modify any existing JSON; each run produces a fresh
  output from the parsed source.
- Add `tool/generate_builtin_units.dart` + `tool/generate_builtin_units_lib.dart`:
  reads `units-parsed.json` and `units-supplementary.json`, merges them into
  `units.json`, then generates `lib/core/domain/data/builtin_units.dart`.
- Add `lib/core/domain/data/units-supplementary.json`: manually-curated file
  (descriptions, aliases, categories, affine unit definitions) that is merged
  over the parsed data at codegen time. Checked in; never overwritten by the
  importer.
- Add `lib/core/domain/data/units.json`: the merged intermediate representation,
  generated by `generate_builtin_units.dart` from `units-parsed.json` +
  `units-supplementary.json`. Checked in as the direct input to codegen.
- Add `lib/core/domain/data/README.md`: documents the four-file pipeline and
  explains which files to edit and how.
- Regenerate `lib/core/domain/data/builtin_units.dart` from `units.json`,
  adding all Phase 5 unit categories.
- Add a new `bit` primitive dimension for digital information storage.
- Add tests for both tool libraries and for all new unit categories.

## Capabilities

### New Capabilities

- `gnu-units-import-pipeline`: Two-stage pipeline (importer + codegen) for
  maintaining the unit database from a GNU Units definitions file via JSON
  intermediates (`units-parsed.json`, `units-supplementary.json`, `units.json`).

_(none — no existing capability specs change their requirements)_

## Impact

- `tool/` — two new tool libraries + two new executables
- `tool/gnu_units/` — directory for the GNU Units definitions file (user-provided)
- `lib/core/domain/data/units-parsed.json` — new file (generated by importer; not checked in)
- `lib/core/domain/data/units-supplementary.json` — new file (manually curated; checked in)
- `lib/core/domain/data/units.json` — new file (generated by codegen; checked in for codegen)
- `lib/core/domain/data/README.md` — new file documenting the pipeline
- `lib/core/domain/data/builtin_units.dart` — regenerated with Phase 5 units
- `test/tool/` — new tests for importer and codegen libraries
- `test/core/domain/data/builtin_units_test.dart` — extended with Phase 5 tests
