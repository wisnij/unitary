## ADDED Requirements

### Requirement: Items within each section group are ordered oldest-to-newest
Within any changelog section group (Added, Changed, Fixed, Documentation,
Other), items SHALL appear in ascending chronological order — oldest change
first, newest change last.  This applies both to entries generated by the
`changelog` subcommand and to manually written changelog entries.

#### Scenario: Multiple commits in the same section appear oldest-first
- **WHEN** two or more commits map to the same section group (e.g., two `feat`
  commits both appear under Added)
- **THEN** the earlier commit's entry appears before the later commit's entry
  in that section

#### Scenario: Single-item sections are unaffected
- **WHEN** a section group contains exactly one item
- **THEN** that item is written as-is (ordering does not change single items)
